## What is Calango

**Calango** is a Python meta-framework for web development with flexible Convention over Configuration (CoC). Think Phoenix Framework or Nest.js, but Python ‚Äî with AI as a first-class citizen and TDD as the path of least resistance.

The mascot is a calango lizard ü¶é wearing nerd glasses and a bow tie. Personality: clever, quick, a little bold ‚Äî but always helpful.

**Tagline:** *The fast, friendly Python web framework.*

## Code conventions ‚Äî follow strictly

### Naming

```python
# Models ‚Äî PascalCase, no suffix
class Order(Base): ...

# Schemas ‚Äî PascalCase + mandatory suffix
class OrderInput(CalangoModel): ...
class OrderOutput(CalangoModel): ...
class OrderUpdate(CalangoModel): ...

# Repositories ‚Äî PascalCase + Repository
class OrderRepository(BaseRepository[Order]): ...

# Services ‚Äî PascalCase + Service
class OrderService(BaseService[OrderRepository]): ...

# Routers ‚Äî snake_case, one file per resource
# app/routers/order.py

# Tests ‚Äî descriptive, document the business rule
async def test_order_with_discount_above_100_should_apply_free_shipping(): ...
```

### Layered architecture ‚Äî dependency order

```
routers ‚Üí services ‚Üí repositories ‚Üí models
```

- **Routers**: receive request, validate schema, delegate to service, return response. Zero logic.
- **Services**: all business rules. Do not touch the database ‚Äî call the repository.
- **Repositories**: only layer that executes queries. No business logic.
- **Models**: structure definition. No business methods.

### CLI conventions (`calango_cli/`)

All terminal output and interactive prompts are centralised in `calango_cli/ui.py`. Command files in `calango_cli/commands/` import only from `calango_cli.ui` ‚Äî never from `rich` or `questionary` directly.

**Interactive pattern** ‚Äî commands use `str | None` for required arguments and fall back to an interactive wizard when the arg is missing and `is_interactive()` is True:

```python
def my_cmd(name: str | None = typer.Argument(None, help="...")) -> None:
    if name is None:
        if not is_interactive():
            print_error("Missing argument 'NAME'.", hint="Run interactively: calango my-cmd")
            raise typer.Exit(1)
        name = ask("Name")
```

**CI safety** ‚Äî flags always skip prompts. `is_interactive()` returns False when stdin or stdout is not a TTY, so scripts and CI pipelines never hang.

**Brand colors for the terminal** ‚Äî use Rich style names from `ui._THEME`:
`calango.primary` (#2CBD6B), `calango.accent` (#E29A2E), `calango.danger` (#E5575C), `calango.muted` (#6A736E), `calango.text` (#A9B2AD).

See `packages/calango-cli/CLAUDE.md` for the full authoring guide.

### What to NEVER do

```python
# ‚ùå Business logic in the router
@router.post("/orders")
async def create_order(data: OrderInput, db: AsyncSession = Depends(get_db)):
    if data.amount > 1000:  # WRONG ‚Äî this belongs in the service
        ...

# ‚ùå Database access directly in the service
class OrderService:
    async def create(self, data: OrderInput, db: AsyncSession):  # WRONG
        await db.execute(...)

# ‚ùå Logic in the model
class Order(Base):
    def calculate_discount(self):  # WRONG ‚Äî this belongs in the service
        ...

# ‚ùå Raw SQL without parameterization
await session.execute(f"SELECT * FROM orders WHERE id = '{id}'")  # SQL INJECTION

# ‚ùå Synchronous library in async context
@router.get("/data")
async def get_data():
    response = requests.get("https://api.example.com")  # BLOCKS THE EVENT LOOP

# ‚ùå Stack trace in production
raise HTTPException(detail=str(exc))  # Use the Calango global handler

# ‚ùå Hardcoded secrets
SECRET_KEY = "my-secret-key"  # Use pydantic-settings + env var

# ‚ùå PII in logs
logger.info(f"Login: {user.email} password={password}")  # NEVER
```

### What to ALWAYS do

```python
# ‚úÖ Router delegates to service
@router.post("/orders")
async def create_order(
    data: OrderInput,
    current_user: User = Depends(get_current_user),
    service: OrderService = Depends(get_order_service),
) -> OrderOutput:
    return await service.create(data, user=current_user)

# ‚úÖ Service uses repository
class OrderService(BaseService[OrderRepository]):
    async def create(self, data: OrderInput, *, user: User) -> Order:
        return await self.repository.create(data, owner_id=user.id)

# ‚úÖ Repository with explicit eager loading
class OrderRepository(BaseRepository[Order]):
    async def get_with_customer(self, id: UUID) -> Order | None:
        result = await self.session.execute(
            select(Order)
            .options(selectinload(Order.customer))
            .where(Order.id == id)
        )
        return result.scalar_one_or_none()

# ‚úÖ Schema separated by operation
class OrderInput(CalangoModel):
    amount: Decimal
    customer_id: UUID

class OrderOutput(CalangoModel):
    id: UUID
    amount: Decimal
    status: str
    created_at: datetime
    customer: CustomerOutput

class OrderUpdate(CalangoModel):
    status: Literal["confirmed", "cancelled"] | None = None
```

## AgentBackend ‚Äî the agent contract

```python
from typing import Protocol, AsyncIterator, runtime_checkable

@runtime_checkable
class AgentBackend(Protocol):
    async def run(self, prompt: str, context: AgentContext) -> AgentResponse: ...
    async def stream(self, prompt: str, context: AgentContext) -> AsyncIterator[str]: ...
    def register_tool(self, fn: ToolDefinition) -> None: ...
    def list_tools(self) -> list[str]: ...
    async def get_memory(self, session_id: str, user_id: str) -> list[dict]: ...
    async def save_memory(self, session_id: str, user_id: str, messages: list[dict]) -> None: ...
    async def startup(self) -> None: ...
    async def shutdown(self) -> None: ...
```

**Default implementation: Agno v2.6+**
- Performance: ~2¬µs per agent, 3.75 KiB per session
- Native FastAPI via `AgentOS.get_app()`
- MCP: consumes and exposes MCP servers
- 23+ LLM providers
- Native human-in-the-loop
- ReliabilityEval for CI

**AgentContext ‚Äî injected automatically:**



## Security ‚Äî OWASP by design

### Posture: secure by default, explicit to override

```python
# All endpoints require auth by default (identity plugin active)
@router.get("/orders")
async def list_orders(current_user: User = Depends(get_current_user)):
    ...

# Explicit public exception ‚Äî visible in the code
@router.get("/status")
@public
async def health():
    ...

# Declarative RBAC
@router.post("/admin/users")
@require_permission("users:admin")
async def create_admin_user(...):
    ...
```

### Security checklist implemented in the framework

**OWASP Top 10 Web 2025:**
- A01 Broken Access Control ‚Üí explicit `@public`, ownership check in repository
- A02 Security Misconfiguration ‚Üí automatic HTTP headers, closed CORS
- A03 Supply Chain ‚Üí committed `uv.lock`, `pip-audit` in CI
- A04 Cryptographic Failures ‚Üí Argon2 for passwords, RS256 for JWT, `secrets` module
- A05 Injection ‚Üí mandatory SQLAlchemy ORM, Ruff rules for raw SQL
- A06 Insecure Design ‚Üí generated SECURITY.md, secure password reset by default
- A07 Authentication Failures ‚Üí rate limit on `/auth`, short JWT expiry, refresh rotation
- A08 Data Integrity ‚Üí mandatory webhook signature verification, idempotency in jobs
- A09 Logging Failures ‚Üí security events logged automatically
- A10 Exceptional Conditions ‚Üí global handler that never exposes stack trace in production

**OWASP API Security 2023:**
- API1 BOLA ‚Üí repository always receives `current_user`, ownership verified
- API3 Mass Assignment ‚Üí schemas separated by operation (Input/Update/Output)
- API4 Resource Consumption ‚Üí mandatory pagination, default rate limiting

**OWASP LLM 2025:**
- LLM01 Prompt Injection ‚Üí guardrails plugin, system/user prompt separation
- LLM02 Sensitive Disclosure ‚Üí PII filter on agent output
- LLM06 Excessive Agency ‚Üí `requires_approval` for destructive actions
- LLM10 Unbounded Consumption ‚Üí token budget via Plans plugin

### Calango custom Ruff rules

```
CL001: Use 'httpx.AsyncClient' instead of 'requests' in async context
CL002: Use 'aiofiles.open' instead of 'open' in async context
CL003: Blocking call inside coroutine
CL020: HTTP call to dynamic URL ‚Äî validate against allowlist
CL030: Hardcoded secret detected
CL031: Weak hash algorithm
CL032: 'random' module used for security
CL040: Raw SQL string interpolation
CL050: Potential PII in log statement
CL010: CPU-bound handler ‚Äî consider background job
```

## Performance ‚Äî what the framework detects and prevents

### N+1 queries

In development, automatic header on every response:
```
X-Calango-Query-Count: 47
X-Calango-N1-Warning: OrderRepository.get_customer (43 similar queries)
```

Threshold: warning at 5+ similar queries, error (500) at 20+ (dev only).

### Event loop

Watchdog detects blocking > 50ms and logs warning with stack trace.
Ruff rules detect synchronous blocking calls inside async functions.

### Connection pool ‚Äî healthy defaults

```python
engine = create_async_engine(
    DATABASE_URL,
    pool_size=10,
    max_overflow=20,
    pool_timeout=30,
    pool_recycle=1800,
    pool_pre_ping=True,
)
```

### Automatic missing index detection

```bash
calango db suggest-indexes
# Analyzes query history, suggests indexes with estimated impact
# Automatically generates Alembic migration
```

### Concurrency primitives

```python
from calango.concurrency import atomic, advisory_lock
from calango.cache import cached

@atomic
async def confirm_payment(order_id: UUID): ...

async with advisory_lock(f"inventory:{product_id}"): ...

@cached(ttl=300, lock=True)  # Prevents cache stampede
async def get_catalog(): ...
```

## Multi-tenancy (calango-multitenancy plugin)

- **Row-level (default):** Postgres RLS ‚Äî isolation at the database level, not the code level
- **Schema-level:** one schema per tenant, migrations via Alembic
- **Database-level:** separate database per tenant (enterprise)

```python
@tenantable  # Adds tenant_id + RLS policy automatically
class Order(Base): ...
```

Resolution: subdomain ‚Üí JWT claim ‚Üí `X-Tenant-ID` header

## CLI commands

```bash
# Scaffold
calango new <name> [--db=postgres|mongo] [--ci=github|gitlab|bitbucket] [--agents]

# Code generation
calango generate resource <name>     # model + schema + repo + service + router + tests
calango generate model <name>
calango generate service <name>
calango generate agent <name>
calango generate version v2

# Database
calango db migrate
calango db seed
calango db rollback
calango db suggest-indexes           # Detects columns without index and generates migration
calango db analyze-indexes           # Detects unused indexes

# Plugins
calango plugin add <name>
calango plugin remove <name>
calango plugin list

# Tests
calango test                         # All tests
calango test --watch                 # Re-run on save
calango test --integration           # Integration only
calango test --evals                 # Agent evals
calango test --lint-names            # Validates descriptive test names
calango test --cov-report=html

# Quality and security
calango check:security [--env=production]
calango check:indexes
calango bench [--compare-baseline]
calango profile GET /api/v1/orders

# AI context
calango context                      # Regenerate CLAUDE.md and equivalents
calango context --check              # Verify it is up to date (CI)
calango context --show               # Display in terminal

# SDK
calango sdk:generate [--lang=typescript|python|kotlin|swift]
calango sdk:check

# Release
calango release                      # Calculate version, generate CHANGELOG, create tag
calango commit                       # Interactive commit assistant

# Setup
calango setup:github                 # Configure labels, protection, environments
calango setup:agile [--tool=linear|jira|github]
```
