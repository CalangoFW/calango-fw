## What is Calango

**Calango** is an **API-first** Python meta-framework for web development with flexible Convention over Configuration (CoC). Think Phoenix Framework or Nest.js, but Python — with AI as a first-class citizen and TDD as the path of least resistance.

The mascot is a calango lizard 🦎 wearing nerd glasses and a bow tie. Personality: clever, quick, a little bold — but always helpful.

**Tagline:** *The fast, friendly Python web framework.*

## Code conventions — follow strictly

### Quality loop — run on every change

Treat the linter and type checker like the test suite: run them after **every**
code modification, not just before committing. The same three gates the CI
enforces must pass locally before you consider any change done:

```bash
uv run ruff check .            # lint + SAST (Ruff S) — must report "All checks passed!"
uv run ruff format --check .   # format — must report all files formatted
uv run ty check packages/      # types — must report "All checks passed!"
uv run pytest packages/        # tests — must be green
calango check:security         # SCA (pip-audit) + SAST (Opengrep) — no findings
```

Fix issues at the source. Use `# ty: ignore[rule]` (not `# type: ignore`) **only**
for genuine type-checker limitations — pydantic-settings env loading, SQLAlchemy
dynamic attributes, Starlette handler variance, or tests that pass invalid types
on purpose — and always with a one-line comment explaining why. Never silence a
real bug with an ignore. The pre-commit hooks run the quality gates; keep them
installed (`uvx pre-commit install`) so nothing reaches CI unchecked.

### Security tooling — SAST + SCA by default

The framework and every generated project ship a three-layer static security gate,
all of it free, OSS, and run **as tools** (never imported into the app — so no
license copyleft touches user code):

| Layer | Tool | License | Where |
|---|---|---|---|
| SAST inline | Ruff `S` (flake8-bandit) | MIT | every `ruff check` |
| SAST deep + custom rules | Opengrep (Semgrep fork) | LGPL-2.1 | CI `security` job, `check:security` |
| SCA (dependency CVEs) | pip-audit (PyPA) | Apache-2.0 | CI `security` job, `check:security` |

The CI `security` job **blocks** on a known CVE with a fix (pip-audit) or any
high-severity Opengrep finding. Calango's own rules (CL040 raw SQL, CL050 PII in
log; more to come) live in `security/opengrep/calango.yml` — these are the
**Opengrep implementation** of the custom `CL0xx` rules, since Ruff has no custom-rule
support. Runtime defense is by design (middleware, validation, rate limit, ownership
checks), not a shipped dependency: in-app RASP options were rejected as defaults
(Aikido Zen is AGPL-3.0, PyRASP is CC-BY-NC NonCommercial). An edge WAF (OWASP Coraza,
Apache-2.0) and a Zen opt-in plugin are possible future additions.

### Naming

```python
# Contexts — PascalCase (domain grouping, e.g. Shop, Accounts, Catalog)
# CLI: calango generate resource Shop.Order
# Directory: app/contexts/shop/

# Models — PascalCase, no suffix
class Order(Base): ...

# Schemas — PascalCase + mandatory suffix
class OrderInput(CalangoModel): ...
class OrderOutput(CalangoModel): ...
class OrderUpdate(CalangoModel): ...

# Repositories — PascalCase + Repository
class OrderRepository(BaseRepository[Order]): ...

# Services — PascalCase + Service
class OrderService(BaseService[OrderRepository]): ...

# Routers — snake_case, one file per resource, lives in app/routers/ (not inside context)
# app/routers/order.py

# Tests — descriptive, document the business rule
async def test_order_with_discount_above_100_should_apply_free_shipping(): ...
```

### Layered architecture — dependency order

```
routers → services → repositories → models
```

- **Routers**: receive request, validate schema, delegate to service, return response. Zero logic.
- **Services**: all business rules. Do not touch the database — call the repository.
- **Repositories**: only layer that executes queries. No business logic.
- **Models**: structure definition. No business methods.

### CLI conventions (`calango_cli/`)

All terminal output and interactive prompts are centralised in `calango_cli/ui.py`. Command files in `calango_cli/commands/` import only from `calango_cli.ui` — never from `rich` or `questionary` directly.

**Interactive pattern** — commands use `str | None` for required arguments and fall back to an interactive wizard when the arg is missing and `is_interactive()` is True:

```python
def my_cmd(name: str | None = typer.Argument(None, help="...")) -> None:
    if name is None:
        if not is_interactive():
            print_error("Missing argument 'NAME'.", hint="Run interactively: calango my-cmd")
            raise typer.Exit(1)
        name = ask("Name")
```

**CI safety** — flags always skip prompts. `is_interactive()` returns False when stdin or stdout is not a TTY, so scripts and CI pipelines never hang.

**Brand colors for the terminal** — use Rich style names from `ui._THEME`:
`calango.primary` (#2CBD6B), `calango.accent` (#E29A2E), `calango.danger` (#E5575C), `calango.muted` (#6A736E), `calango.text` (#A9B2AD).

See `packages/calango-cli/CLAUDE.md` for the full authoring guide.

### GitHub Actions — keep current

All workflow files must include the top-level env var:

```yaml
env:
  FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: true
```

This ensures every action runs on Node.js 24 regardless of the action's pinned version. GitHub removes Node.js 20 from runners in September 2026. Never remove this env var and never add a workflow file without it.

### What to NEVER do

```python
# ❌ Business logic in the router
@router.post("/orders")
async def create_order(data: OrderInput, db: AsyncSession = Depends(get_db)):
    if data.amount > 1000:  # WRONG — this belongs in the service
        ...

# ❌ Database access directly in the service
class OrderService:
    async def create(self, data: OrderInput, db: AsyncSession):  # WRONG
        await db.execute(...)

# ❌ Logic in the model
class Order(Base):
    def calculate_discount(self):  # WRONG — this belongs in the service
        ...

# ❌ Raw SQL without parameterization
await session.execute(f"SELECT * FROM orders WHERE id = '{id}'")  # SQL INJECTION

# ❌ Synchronous library in async context
@router.get("/data")
async def get_data():
    response = requests.get("https://api.example.com")  # BLOCKS THE EVENT LOOP

# ❌ Stack trace in production
raise HTTPException(detail=str(exc))  # Use the Calango global handler

# ❌ Hardcoded secrets
SECRET_KEY = "my-secret-key"  # Use pydantic-settings + env var

# ❌ PII in logs
logger.info(f"Login: {user.email} password={password}")  # NEVER

# ❌ Cross-context internal import — breaks domain boundaries
from app.contexts.catalog.models.product import Product  # access via public API instead

# ❌ Flat resource generation (no context)
# calango generate resource Order  → WRONG (missing context)
# calango generate resource Shop.Order  → CORRECT
```

### What to ALWAYS do

```python
# ✅ Router delegates to service
@router.post("/orders")
async def create_order(
    data: OrderInput,
    current_user: User = Depends(get_current_user),
    service: OrderService = Depends(get_order_service),
) -> OrderOutput:
    return await service.create(data, user=current_user)

# ✅ Service uses repository
class OrderService(BaseService[OrderRepository]):
    async def create(self, data: OrderInput, *, user: User) -> Order:
        return await self.repository.create(data, owner_id=user.id)

# ✅ Repository with explicit eager loading
class OrderRepository(BaseRepository[Order]):
    async def get_with_customer(self, id: UUID) -> Order | None:
        result = await self.session.execute(
            select(Order)
            .options(selectinload(Order.customer))
            .where(Order.id == id)
        )
        return result.scalar_one_or_none()

# ✅ Schema separated by operation
class OrderInput(CalangoModel):
    amount: Decimal
    customer_id: UUID

class OrderOutput(CalangoModel):
    id: UUID
    amount: Decimal
    status: str
    created_at: datetime
    customer: CustomerOutput

class OrderUpdate(CalangoModel):
    status: Literal["confirmed", "cancelled"] | None = None

# ✅ Router imports from context public API (never from context internals)
from app.contexts.shop import OrderService, OrderRepository, OrderInput, OrderOutput

# ✅ Cross-context dependencies use the public API only
from app.contexts.accounts import UserOutput  # ✅ public API
# from app.contexts.accounts.models.user import User  # ❌ internal
```

## AgentBackend — the agent contract

```python
from typing import Protocol, AsyncIterator, runtime_checkable

@runtime_checkable
class AgentBackend(Protocol):
    async def run(self, prompt: str, context: AgentContext) -> AgentResponse: ...
    async def stream(self, prompt: str, context: AgentContext) -> AsyncIterator[str]: ...
    def register_tool(self, fn: ToolDefinition) -> None: ...
    def list_tools(self) -> list[str]: ...
    async def get_memory(self, session_id: str, user_id: str) -> list[dict]: ...
    async def save_memory(self, session_id: str, user_id: str, messages: list[dict]) -> None: ...
    async def startup(self) -> None: ...
    async def shutdown(self) -> None: ...
```

**Default implementation: Agno v2.6+**
- Performance: ~2µs per agent, 3.75 KiB per session
- Native FastAPI via `AgentOS.get_app()`
- MCP: consumes and exposes MCP servers
- 23+ LLM providers
- Native human-in-the-loop
- ReliabilityEval for CI

**AgentContext — injected automatically:**

```python
@dataclass
class AgentContext:
    user_id: str | None
    tenant_id: str | None
    session_id: str
    request_id: str
    ip_address: str
    permissions: list[str]
    plan: str | None
    metadata: dict[str, Any]
```

**Usage:**

```python
from calango.agents import AgentRouter

router = AgentRouter(prefix="/agents")

@router.agent("/support", streaming=True, guardrails=True)
async def support_agent():
    """Support agent with access to real account data."""

@router.tool(permissions=["orders:read"])
async def get_orders(context: AgentContext, status: str | None = None):
    """Tool always scoped to the authenticated user — no BOLA by design."""
    return await OrderRepository.list_by_user(user_id=context.user_id)

@router.tool(requires_approval=True)
async def request_refund(context: AgentContext, order_id: str, reason: str):
    """Human-in-the-loop: pauses, notifies, waits for human approval."""
    ...
```

## Security — OWASP by design

### Posture: secure by default, explicit to override

```python
# All endpoints require auth by default (identity plugin active)
@router.get("/orders")
async def list_orders(current_user: User = Depends(get_current_user)):
    ...

# Explicit public exception — visible in the code
@router.get("/status")
@public
async def health():
    ...

# Declarative RBAC
@router.post("/admin/users")
@require_permission("users:admin")
async def create_admin_user(...):
    ...
```

### Security checklist implemented in the framework

**OWASP Top 10 Web 2025:**
- A01 Broken Access Control → explicit `@public`, ownership check in repository
- A02 Security Misconfiguration → automatic HTTP headers, closed CORS
- A03 Supply Chain → committed `uv.lock`, `pip-audit` in CI
- A04 Cryptographic Failures → Argon2 for passwords, RS256 for JWT, `secrets` module
- A05 Injection → mandatory SQLAlchemy ORM, Ruff rules for raw SQL
- A06 Insecure Design → generated SECURITY.md, secure password reset by default
- A07 Authentication Failures → rate limit on `/auth`, short JWT expiry, refresh rotation
- A08 Data Integrity → mandatory webhook signature verification, idempotency in jobs
- A09 Logging Failures → security events logged automatically
- A10 Exceptional Conditions → global handler that never exposes stack trace in production

**OWASP API Security 2023:**
- API1 BOLA → repository always receives `current_user`, ownership verified
- API3 Mass Assignment → schemas separated by operation (Input/Update/Output)
- API4 Resource Consumption → mandatory pagination, default rate limiting

**OWASP LLM 2025:**
- LLM01 Prompt Injection → guardrails plugin, system/user prompt separation
- LLM02 Sensitive Disclosure → PII filter on agent output
- LLM06 Excessive Agency → `requires_approval` for destructive actions
- LLM10 Unbounded Consumption → token budget via Plans plugin

### Calango custom security rules (Opengrep)

Ruff has no custom-rule support, so Calango's own `CL0xx` rules are authored as
Opengrep YAML in `security/opengrep/calango.yml`. Shipped so far: **CL040** and
**CL050**. The rest below are planned — add each as an Opengrep rule with a test
fixture proving it fires, and confirm it does not false-positive on `packages/`.

```
CL001: Use 'httpx.AsyncClient' instead of 'requests' in async context
CL002: Use 'aiofiles.open' instead of 'open' in async context
CL003: Blocking call inside coroutine
CL010: CPU-bound handler — consider background job
CL020: HTTP call to dynamic URL — validate against allowlist
CL030: Hardcoded secret detected
CL031: Weak hash algorithm
CL032: 'random' module used for security
CL040: Raw SQL string interpolation                 ✅ shipped
CL050: Potential PII in log statement               ✅ shipped
```

(Note: hardcoded-secret/weak-hash/random-for-security overlap with Ruff `S` rules
already enabled — prefer Ruff `S` where it covers the case, reserve `CL0xx` for
Calango-specific patterns Ruff can't express.)

## Performance — what the framework detects and prevents

### N+1 queries

In development, automatic header on every response:
```
X-Calango-Query-Count: 47
X-Calango-N1-Warning: OrderRepository.get_customer (43 similar queries)
```

Threshold: warning at 5+ similar queries, error (500) at 20+ (dev only).

### Event loop

Watchdog detects blocking > 50ms and logs warning with stack trace.
Ruff rules detect synchronous blocking calls inside async functions.

### Connection pool — healthy defaults

```python
engine = create_async_engine(
    DATABASE_URL,
    pool_size=10,
    max_overflow=20,
    pool_timeout=30,
    pool_recycle=1800,
    pool_pre_ping=True,
)
```

### Automatic missing index detection

```bash
calango db suggest-indexes
# Analyzes query history, suggests indexes with estimated impact
# Automatically generates Alembic migration
```

### Concurrency primitives

```python
from calango.concurrency import atomic, advisory_lock
from calango.cache import cached

@atomic
async def confirm_payment(order_id: UUID): ...

async with advisory_lock(f"inventory:{product_id}"): ...

@cached(ttl=300, lock=True)  # Prevents cache stampede
async def get_catalog(): ...
```

## Multi-tenancy (calango-multitenancy plugin)

- **Row-level (default):** Postgres RLS — isolation at the database level, not the code level
- **Schema-level:** one schema per tenant, migrations via Alembic
- **Database-level:** separate database per tenant (enterprise)


Resolution: subdomain → JWT claim → `X-Tenant-ID` header

## CLI commands

```bash
# Scaffold
calango new <name> [--db=postgres|mongo] [--ci=github|gitlab|bitbucket] [--agents]

# Code generation
calango generate resource <Context.Name>  # model + schema + repo + service + router + tests
calango generate model <name>
calango generate service <name>
calango generate agent <name>
calango generate version v2

# Database
calango db migrate
calango db seed
calango db rollback
calango db suggest-indexes           # Detects columns without index and generates migration
calango db analyze-indexes           # Detects unused indexes

# Plugins
calango plugin add <name>
calango plugin remove <name>
calango plugin list

# Tests
calango test                         # All tests
calango test --watch                 # Re-run on save
calango test --integration           # Integration only
calango test --evals                 # Agent evals
calango test --lint-names            # Validates descriptive test names
calango test --cov-report=html

# Quality and security
calango check:security [--env=production]
calango check:indexes
calango bench [--compare-baseline]
calango profile GET /api/v1/orders

# AI context
calango context                      # Regenerate CLAUDE.md and equivalents
calango context --check              # Verify it is up to date (CI)
calango context --show               # Display in terminal

# SDK
calango sdk:generate [--lang=typescript|python|kotlin|swift]
calango sdk:check

# Release
calango release                      # Calculate version, generate CHANGELOG, create tag
calango commit                       # Interactive commit assistant

# Setup
calango setup:github                 # Configure labels, protection, environments
calango setup:agile [--tool=linear|jira|github]
```
